# ファイル作成ルール

## 重要: 一時ファイル・テストファイルの作成場所

**絶対に `_lib/` や実際のプログラムフォルダに一時ファイルを作らない！**

### 一時ファイル・テストファイルを作る場所
- ✅ `temp/` フォルダ
- ✅ `for_claude/` フォルダ

### NG: 絶対に作らない場所
- ❌ `_lib/` フォルダ
- ❌ その他の実際のプログラムフォルダ

### 理由
- ユーザーがgitで管理しているフォルダ
- exe組み込み対象のフォルダに不要なファイルを入れない

### 例
- テストファイル → `for_claude/test_xxx.py` または `temp/test_xxx.py`
- 一時データ → `temp/temp_data.py`
- 検証スクリプト → `for_claude/verify_xxx.py`

## 本番ファイルの編集
ユーザーが明示的に指示した本番ファイルの編集のみ行う。
